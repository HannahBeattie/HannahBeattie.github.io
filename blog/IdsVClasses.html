This post unpacks how, why and when we might use ids and classes

First, a little context; What is HTML? What is CSS? Who is my real father?
Well, perhaps I can answer two of those questions...
HTML, or 'Hyper Text Markup Language' is a language used to write the content of a webpage.
If we were making a movie, we could call HTML 'the script' ; It could contain scene headings, dialogue and notes about the film's characters.
Like most scripts, HTML follows a specific structure, which means a computer knows how to 'read' and interpret it.
Reading our movie's script would give us a good idea of the plot, genre and setting, but what about the style? Giving our script to Wes Anderson or Quentin Tarantino might render very diffirent visual outcomes.
That's where CSS comes into play. CSS or Cascading Style Sheets is the language used to describe how HTML elements are displayed on screen. Their language is not unlike that of the set designers, cinematographers, make-up artists and prop departments who would create stunning visuals for our movie...although we would probably leave the bulk of the visual affects to our faithful collegues in Javiscript!
As the name suggest, our CSS document would be a 'style sheet' with a list of visual and formatting instructions to be applied to our HTML file.
We can link as many HTML files as we would like to the same CSS file, to express the same style rules to multiple documents.

WOW! Neat. But what if we only wanted to make the first two (p1) sentences of our HTML doc rainbow? It's all class baby!
In CSS we use selectors to specify which elements of an HTML doc we want to add style attributes to. Two of the most commonly used selectors are 'class' and 'id'.
The 'class' selector is used to select a few diffirent elements in an HTML doc, grouping them under one label so that CSS knows to apply style changes to just those elements.
Think of 'class' as a huge, brightly-coloured highlighter pen; If we highlighted all of the words in the first (p1) paragraph of our diagram, it would be super easy (barely an inconvenience!) to tell CSS to only apply our styles the highlighted sections.

But what about ids?
Well, an 'id selector' is like a class selector's exclusive, fancy, celebrity cousin. While they too specify which content in an HTML document we want to apply style elements to, they will only specify a single element.
For instance, in the image below we are assigning the id of 'red' to the last paragraph (p2) of the HTML doc.

We are then telling CSS: Please make everything with the id of red red-coloured. CSS obliges, and, voila!; The last paragraph is red.
Let's recap!
A 'class' selector should be applied when there are multiple areas (or elements) you would like change, while a 'id' selector should be used when you would like to apply particular styles to just one element.
